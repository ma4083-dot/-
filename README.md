## ✅ طريقة الرفع على GitHub Pages

### الخطوة 1 — أنشئ مستودع جديد
- اذهب إلى [github.com/new](https://github.com/new)
- اختر اسم للمستودع (مثلاً: `elitelab`)
- اجعله **Public** (مهم لـ GitHub Pages المجاني)
- لا تضف أي ملفات تلقائية (README, .gitignore)

### الخطوة 2 — ارفع الملفات
ارفع **الملفات الثلاثة** التالية في نفس المجلد الجذر:
- `index.html`
- `sw.js`
- `README.md`

**عن طريق واجهة GitHub:**
1. داخل المستودع، اضغط **Add file → Upload files**
2. اسحب الملفات الثلاثة معاً
3. اضغط **Commit changes**

### الخطوة 3 — فعّل GitHub Pages
1. اذهب إلى **Settings** (في المستودع)
2. من القائمة الجانبية: **Pages**
3. تحت **Source**: اختر **Deploy from a branch**
4. الـ Branch: **main** ← **/ (root)**
5. اضغط **Save**

### الخطوة 4 — افتح التطبيق
بعد دقيقة أو دقيقتين، سيكون التطبيق متاحاً على:
```
https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
```

## 🌐 كيف يعمل الأوف لاين؟

| الحالة | ما يحدث |
|--------|---------|
| **أول زيارة (أونلاين)** | يحمّل التطبيق ويحفظ كل شيء في الكاش |
| **زيارة تالية (أوف لاين)** | يعمل من الكاش مباشرةً بدون إنترنت |
| **البيانات (Google Sheets)** | تُحفظ محلياً وتُزامن تلقائياً عند عودة الإنترنت |
| **CDN (مكتبات JS/CSS)** | محفوظة في الكاش من أول اتصال |

---

## ⚠️ ملاحظات مهمة

- يجب أن يكون `sw.js` في **نفس مجلد** `index.html` حتى يعمل الأوف لاين
- المتصفح يحتاج **HTTPS** لتسجيل Service Worker — GitHub Pages يوفر HTTPS تلقائياً ✅
- عند تحديث التطبيق، غيّر رقم الإصدار في `sw.js` (`CACHE = 'elitelab-v5'`) ليُجبر المتصفح على تحميل النسخة الجديدة
